# ChurnAI

A customer churn prediction API built with **FastAPI** and a **Random Forest** classifier, trained on the [Telco Customer Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn).

Given a customer's account details (contract type, tenure, charges, services subscribed, etc.), ChurnAI predicts whether that customer is likely to churn, along with a probability score.

## How it works

1. **Data cleaning** — Handles missing `TotalCharges` values for brand-new customers (tenure = 0), and encodes all categorical fields (contract type, payment method, etc.) into numeric form.
2. **Model training** — A `RandomForestClassifier` is trained on ~80% of the data, with `class_weight="balanced"` and a capped `max_depth` to reduce overfitting and improve detection of the minority (churn) class.
3. **Serving** — FastAPI loads the trained model once at startup and exposes a `/predict` endpoint that takes structured customer data and returns a churn prediction + probability.

## Why Random Forest, and why these settings

The dataset is imbalanced — roughly 73% of customers stay and 27% churn. A model that's only optimized for accuracy can hit ~73% by always predicting "stays," while being useless at catching actual churners.

Because of this, the model is tuned and evaluated primarily on **recall for the churn class**, not raw accuracy:

| Metric | Value |
|---|---|
| Accuracy | ~0.79 |
| Churn recall | ~0.72 |
| Churn precision | ~0.58 |

Recall is prioritized because in a real churn use case, missing an at-risk customer is typically more costly than a false alarm — a false positive just means an unnecessary retention email, while a missed churner is a lost customer.

## Tech stack

- **FastAPI** — API framework
- **scikit-learn** — model training (RandomForestClassifier)
- **pandas** — data loading and preprocessing

## Running it locally

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Train the model (only needs to be run once, or whenever the data changes)
python train_model.py

# 3. Start the API
uvicorn main:app --reload
```

The API will be running at `http://127.0.0.1:8000`. Interactive docs (auto-generated by FastAPI) are available at `http://127.0.0.1:8000/docs`, where you can test `/predict` directly in the browser.

```json

{
  "gender": "Female",
  "SeniorCitizen": 0,

  "Dependents": "No",

  "MultipleLines": "No phone service",
  "InternetService": "DSL",
```
